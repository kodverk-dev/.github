 
# 📁 Repository Naming Conventions

 Соглашение об именовании репозиториев организации.   
 
## Схема именования

```
[тип]-[название]
[тип]-[клиент]-[название]        ← для клиентских проектов
[тип]-[название]--[подсервис]    ← для монорепо / отдельных сервисов
```

---

## Типы репозиториев

| Префикс   | Назначение                              | Пример                    |
|-----------|-----------------------------------------|---------------------------|
| `client-` | Клиентские проекты                      | `client-sevas-biju`       |
| `own-`    | Собственные продукты и стартапы         | `own-price-tracker`       |
| `tool-`   | Внутренние инструменты                  | `tool-tg-commander`       |
| `lib-`    | Переиспользуемые библиотеки / пакеты    | `lib-ui-kit`              |
| `infra-`  | Инфраструктура, DevOps, конфиги         | `infra-vps-setup`         |
| `bot-`    | Telegram и любые другие боты            | `bot-notifications`       |
| `tmpl-`   | Шаблоны и стартеры проектов             | `tmpl-nextjs-starter`     |

---

## Правила написания

- **Только строчные буквы** — никаких заглавных
- **Только дефисы** — никаких `_`, пробелов, точек
- **Максимум 40 символов** — GitHub обрезает длинные имена в UI
- **Без дат в названии** — для этого есть теги и релизы
- **Kebab-case везде** — `my-cool-project`, не `myCoolProject`
- **Подсервисы через двойной дефис** `--` — визуально отделяет сервис от проекта

## Примеры

### Клиентские проекты

```
client-saleshop              ← монолит / небольшой проект
client-saleshop--api         ← отдельный бэкенд
client-saleshop--admin       ← отдельная админка
client-saleshop--bot         ← бот в рамках проекта клиента
client-hotel-sunrise
client-hotel-sunrise--landing
```

### Собственные продукты

```
own-price-tracker
own-ai-writer
own-ai-writer--bot
own-ai-writer--web
```

### Инструменты и инфра

```
tool-tg-commander
tool-deploy-scripts
infra-vps-setup
infra-docker-base
```

### Библиотеки и шаблоны

```
lib-ui-kit
lib-api-client
tmpl-nextjs-starter
tmpl-bot-starter
```

## Частые вопросы

**Q: Куда отнести лендинг клиента без бэкенда?**  
A: `client-[название]` — подсервис не нужен, это самостоятельный проект.

**Q: Куда отнести общий UI-кит, который используется в нескольких проектах?**  
A: `lib-ui-kit` — именно для этого существует префикс `lib-`.

**Q: Бот является частью клиентского проекта — куда его?**  
A: `client-[название]--bot` — подсервис в рамках проекта клиента.

**Q: Бот является самостоятельным внутренним инструментом?**  
A: `bot-[название]` или `tool-[название]` — на усмотрение команды.
